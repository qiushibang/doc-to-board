# 画板模板说明

> doc-to-board 的画板模板定义。定义不同画板类型的结构和视觉规范。

## 可用模板

### 1. outline-tree（大纲树形图）

**适用场景**：单文档整体结构概览
**布局**：自上而下的树形结构
**特点**：
- 展示完整的文档大纲层级（H1 → H2 → H3）
- 每个节点显示标题 + 简要摘要
- 难点用特殊标记
- 章节间逻辑关系用连线标注

### 2. focus-section（聚焦章节图）

**适用场景**：用户只想关注文档的某一部分
**布局**：以目标章节为中心的放射/树形图
**特点**：
- 目标章节完整展开
- 其他章节折叠显示（仅标题，无摘要）
- 目标章节的子节点完整展示要点
- 上下文关联用虚线连接

### 3. concept-map（概念关系图）

**适用场景**：展示文档核心概念之间的关系
**布局**：网状/自由布局
**特点**：
- 核心概念为中心节点
- 概念间关系用有向边连接
- 每条边标注关系类型
- 适合理论性/学术性文档

### 4. multi-doc-comparison（多文档对比图）

**适用场景**：2-4 个文档的结构对比
**布局**：左右并列 + 中间关联
**特点**：
- 每个文档独立区域
- 跨文档关联用彩色线连接
- 共同主题高亮
- 冲突关系用红色标记

### 5. multi-doc-overview（多文档总览图）

**适用场景**：3+ 个文档的全局关系图
**布局**：中心辐射/集群布局
**特点**：
- 文档为一级节点
- 章节为二级节点
- 跨文档关联简化（只显示强关联）
- 适合大范围文档梳理

## 模板选择指南

| 条件 | 推荐模板 |
|------|---------|
| 单文档，看全貌 | outline-tree |
| 单文档，关注部分 | focus-section |
| 单文档，理论/概念密集 | concept-map |
| 2-4 文档，对比关联 | multi-doc-comparison |
| 3+ 文档，全局梳理 | multi-doc-overview |

## 用户交互

选择模板后，询问用户：
1. **配色风格**（默认 Monochrome，可选其他）
2. **信息密度**（简洁/标准/详细）
3. **是否标注逻辑关系**（默认是）
